# 메일 작성·고도화·발송 (Email Send)

플러그 의뢰 단계에 맞는 템플릿을 제안하고, 사용자 초안을 다듬어 Gmail로 보내고, 의뢰 이력에 기록합니다.
**사람에게 도달하는 작업이므로 발송은 항상 사용자 명시 승인 후에만 진행합니다.**

---

## 언제 사용하나

| 트리거 예시 | 진입 모드 |
|---|---|
| "메일 보내줘", "[회사명] 메일", "이 의뢰 메일" | 전체 절차 (A→B→C) |
| "답장 초안", "팔로업 메일" | 전체 절차 (스레드 유지) |
| "견적 송부 메일", "이 의뢰 다음 단계 메일" | 전체 절차 (단계 기반 템플릿 우선) |
| "이 메일 다듬어줘", "초안 정리해줘" | 다듬기-only 모드 |

---

## 입력

- **의뢰 ID** (있을 때): 플러그에서 수신자·단계·첨부·이력을 자동 조회.
- **수신자**: 이름 + 이메일. 의뢰 ID로 조회하면 자동 채워짐.
- **목적**: 없으면 단계 기반으로 추론해 사용자에게 확인.
- **첨부**: Google Drive 파일 ID 또는 플러그 의뢰 `fileSet[].file` presigned URL.
- **거친 초안** (선택): 붙여넣으면 고도화 모드로 진입.

값이 비었으면 추측하지 말고 한 번만 모아서 묻는다.

---

## 핵심 기능 3가지

### (A) 단계 기반 템플릿 추천

1. `GET /v1/inquiry/{id}` → `status.name` 확인.
2. `GET /v1/inquiry/status` → 워크스페이스 단계 목록 전체 조회.
3. `skills/email-send/templates/` 아래 파일의 프론트매터 `stage:` 배열과 매칭.
   - 정확 매치 우선 → 키워드 부분 매치 순.
4. 후보 1–3개를 번호 목록으로 사용자에게 보여주고 선택받는다.

```
아래 템플릿 중 하나를 골라주세요. 직접 쓰시려면 "직접 쓸게"라고 하세요.

1. [견적송부] 30-견적송부.md — 견적서를 첨부해 보내는 표준 메일
2. [팔로업]   40-팔로업.md   — 견적 발송 후 3–5일 뒤 답변 확인
3. [계약안내] 50-계약안내.md — 구두 수주 후 계약서 서명 안내
```

### (B) 메일 고도화·다듬기

- 사용자가 거친 초안을 붙여넣으면 한국어 비즈니스 톤으로 개선.
- **사용자 의도 추가·변경 금지** — 내용 추가가 필요하면 물어본 뒤 반영.
- 구조: 인사 → 맥락(이전 대화·문의 인용) → 본론 → CTA → 서명.
- 너무 길지 않게 — 핵심을 3단락 이내로.
- 누락된 정보(수신자 이름, 날짜 등)는 `{{...}}` 표시로 남기고 사용자에게 묻는다.

### (C) Gmail 발송 + 플러그 이력 기록

- 사용자 승인 후 `gmail` MCP `send` 호출.
- 답장이면 `In-Reply-To` / `threadId`로 스레드 유지.
- 발송 완료 후 `POST /v1/inquiry/{id}/text_history` 로 메모 기록.

## 절차

### 1단계: 컨텍스트 수집

```bash
# 의뢰 본문 (status, client, fileSet, fieldSet 포함)
scripts/pluuug.py GET /v1/inquiry/{id}

# 고객 정보 (수신자 이메일, 담당자, 직책)
scripts/pluuug.py GET /v1/client/{id}

# 플러그 메일 발신 이력 (이전 메일 톤·미해결 질문)
scripts/pluuug.py GET /v1/inquiry/{id}/email_history
scripts/pluuug.py GET /v1/inquiry/{id}/email_reply_history

# 워크스페이스 단계 목록
scripts/pluuug.py GET /v1/inquiry/status
```

- Gmail MCP로 같은 도메인 이전 스레드 검색 → `In-Reply-To` 가능 여부 확인.
- 답장이면 원본 스레드 ID 확보.

### 2단계: 템플릿 매칭

- 의뢰 `status.name`으로 `templates/` 후보 1–3개 도출.
- 번호 목록으로 사용자에게 보여주고 선택받기.
- 사용자가 "직접 쓸게"라고 하면 빈 구조(인사→본론→CTA→서명)만 제공.

### 3단계: 변수 채우기

- 플러그 데이터로 자동 채움: `{{회사명}}`, `{{담당자}}`, `{{의뢰명}}`.
- 비어 있는 변수는 **한 번에 모아서** 사용자에게 묻는다.
- 발송 직전까지 `{{...}}` 잔재가 남아 있으면 발송 차단.

### 4단계: 고도화

- 선택한 템플릿(또는 사용자 초안)을 한국어 비즈니스 톤으로 다듬기.
- 인사 → 맥락 → 본론 → CTA → 서명 순서 유지.
- 첨부가 있으면 본문에 파일명·Drive 링크 명시.
- 미팅 제안이 목적이면 `google_calendar` MCP로 사용자 빈 슬롯 1–3개 본문에 포함.

### 5단계: 사용자 승인

전체 메일을 아래 형식으로 출력한 뒤 명시 승인을 기다린다.

```
받는 사람: 홍길동 <gildong@example.com>
참    조: (없음)
숨은참조: (없음)
제    목: Re: [ABC솔루션] 도입 문의 — 견적서 안내

안녕하세요, 홍길동님.
...본문...

감사합니다.
[서명]

[첨부] 견적서_ABC솔루션_20260521.pdf (Drive)
```

**→ 보내 / 수정 / 취소**

승인 문구 없이는 절대 send를 호출하지 않는다.

### 6단계: 발송

```bash
# gmail MCP send 호출 (답장이면 threadId 포함)
```

- 답장이면 원본 스레드에 이어 보낸다 — 새 스레드 생성 금지.
- 첨부가 Drive 파일이면 공유 URL 또는 직접 첨부 방식 중 선택.

### 7단계: 플러그 이력 기록

```bash
scripts/pluuug.py POST /v1/inquiry/{id}/text_history \
  --json '{"content":"메일 발송: [제목] / 수신: [수신자] / [요약 1줄]"}'
```

- 발송 결과(메시지 ID, 시각)를 사용자에게 짧게 보고.
- 팔로업이 필요한 메일이면 "N일 뒤 답장 없으면 알림 받을까요?" 제안.

| 파일 | 대상 단계 (예시) | 용도 | 톤 |
|---|---|---|---|

| `templates/20-미팅제안.md` | 1차상담, 상담중 | 온/오프라인 미팅 일정 제안 | friendly |
| `templates/30-견적송부.md` | 견적발송, 견적 | 견적서 첨부 발송 | formal |
| `templates/40-팔로업.md` | 견적발송, 팔로업 | 견적 발송 후 3–5일 뒤 답변 확인 | friendly |
| `templates/50-계약안내.md` | 계약대기, 계약 | 구두 수주 후 계약서 서명 안내 | formal |
| `templates/60-수주감사.md` | 수주 | 수주 감사 + 킥오프 일정 조율 | friendly |
| `templates/70-보류재개타진.md` | 보류 | 보류 후 N주 뒤 재개 타진 | friendly |

템플릿 매칭 알고리즘:
1. `status.name` == `stage:` 배열 항목 (정확 매치)
2. `status.name`이 `stage:` 항목을 포함하거나 포함됨 (부분 매치)
3. 매치 없으면 사용자에게 직접 선택 요청

## 다듬기-only 모드

의뢰 컨텍스트 없이 메일 본문만 다듬을 때 사용한다.

**진입 조건**: 사용자가 메일 본문을 붙여넣고 "다듬어줘" / "고쳐줘" / "정리해줘"라고 함.

**동작**:
- 의뢰 ID·수신자·발송 여부 묻지 않는다.
- 입력 본문을 한국어 비즈니스 톤으로 개선해 바로 출력.
- 사용자 의도는 그대로 — 내용 추가·삭제 없이 표현·구조만 정리.
- 비어 있는 정보(`{{...}}`)는 표시만 하고 묻지 않는다.
- 발송 여부는 사용자가 명시할 때만 이어서 진행.

**출력 형식**:

```
## 다듬은 메일

제목: [제목]

[본문]

---
주요 변경: 단락 재구성, 경어체 통일, CTA 명확화
```

## 자사 정보 자동 적용 (선택)

`sales-setup`이 저장한 `profile.json`이 있으면 메일 작성 시 자동 참조합니다.

- `me.signature` → 모든 발송 메일의 서명 (영업 담당자 개별 서명. `team[]`에서 active 멤버 기준)
- `voice.tone` → 톤 결정 (`formal_business` / `friendly` / `professional`)
- `company.displayName` · `brand.strengths` · `brand.differentiation` → 자기소개 단락 (첫 응답·미팅 제안)
- `company.homepage` → 인용 / 링크

**메일 본문 작성 시** `{{me.name}}`, `{{me.signature}}`, `{{client.companyName}}` 같은 표준 변수로 채워지며, 자세한 변수 명세는 `skills/sales-setup/references/template_variables.md` 참조.

조회: `skills/sales-setup/scripts/profile.py --show`. 프로필이 없으면 사용자에게 한 번 묻거나 일반 톤으로 작성합니다.

## 가드레일

| 규칙 | 내용 |
|---|---|
| 자동 발송 금지 | 사용자 "보내" 신호 없이 `send` 호출 금지. 이전 대화 승인은 그 1건에만 유효. |
| 수신자 검증 | 이메일이 비었거나 형식 이상(`@` 미포함 등)이면 발송 거부 + 사용자 확인. |
| CC/BCC 동의 | 내부 인원 CC/BCC 추가 시 사용자 동의 필수. |
| 첨부 내용 보고 | 견적서·내부 메모 첨부 시 발송 전 "어떤 정보 포함" 한 줄 보고. |
| 중복 발송 경고 | 같은 수신자에게 24시간 내 비슷한 목적 메일 재발송 시 경고. |
| 변수 미해결 차단 | 본문에 `{{...}}` 잔재가 있으면 발송 막고 사용자에게 알린다. |
| Drive 권한 | 첨부 Drive 파일 공유 권한이 닫혀 있으면 경고만 — 자동 권한 변경 금지. |

## 다른 스킬과의 연계

### quote-writing → email-send

견적서 작성 완료 후 "메일로 보내줘"가 나오면 이 스킬로 인계.
- `quote-writing`이 반환한 Drive 링크 + 합계 + 유효기간을 본문에 자동 인용.
- 템플릿 `30-견적송부.md`를 우선 제안.

```
[연계 컨텍스트]
견적서: 견적서_ABC솔루션_20260521.pdf (Drive: https://drive.google.com/...)
합계: 12,000,000원 (부가세 별도)
유효기간: 2026-06-04
```

### call-summary → email-send

통화 요약 후 "후속 메일 보내줘"가 나오면 이 스킬로 인계.
- 통화에서 논의된 다음 액션을 CTA에 반영.
- 답장 스레드가 있으면 같은 스레드에 이어 발송.

### call-prep → email-send

미팅 전 자료 준비 후 "미팅 제안 메일 보내줘"가 나오면 이 스킬로 인계.
- `call-prep` 결과의 담당자 이름·직책을 인사말에 자동 반영.
- 템플릿 `20-미팅제안.md`를 우선 제안, `google_calendar` 슬롯 포함.

### 플러그 API 참조 엔드포인트 요약

| 목적 | 엔드포인트 | 방식 |
|---|---|---|
| 의뢰 상세 조회 | `/v1/inquiry/{id}` | GET |
| 워크스페이스 단계 목록 | `/v1/inquiry/status` | GET |
| 고객 정보 조회 | `/v1/client/{id}` | GET |
| 메일 발신 이력 | `/v1/inquiry/{id}/email_history` | GET |
| 메일 수신 이력 | `/v1/inquiry/{id}/email_reply_history` | GET |
| 발송 이력 메모 기록 | `/v1/inquiry/{id}/text_history` | POST |
