# 액션 아이템 예시 — Todo 매핑 패턴

콜에서 나온 발화를 Pluuug Todo body로 변환하는 패턴을 정리합니다.
좋은 액션(구체적·검증 가능)과 나쁜 액션(지나치게 추상적)을 비교합니다.

---

## M — 미팅 (Meeting)

**Example 1**
콜 발화: "다음주 수요일 오후 2시에 PoC 데모 진행하기로 했어요"
Todo body:
```json
{
  "inquiry": "<inquiry_id>",
  "type": "M",
  "dueDate": "2026-05-27",
  "dueTime": "14:00",
  "action": "PoC 데모 미팅 진행",
  "actionHistory": "고객 측 기술팀 참석, 실제 데이터 연동 시연 예정"
}
```

**Example 2**
콜 발화: "계약 조건 최종 합의를 위해 이번 주 금요일에 만나자고 했어요"
Todo body:
```json
{
  "inquiry": "<inquiry_id>",
  "type": "M",
  "dueDate": "2026-05-23",
  "action": "계약 조건 최종 합의 미팅",
  "actionHistory": "가격·SLA 조항 확정 목적, 의사결정권자 참석 요청"
}
```

| 구분 | 예시 |
|------|------|
| 나쁜 액션 (지나치게 추상적) | "미팅 잡기", "고객 만나기" |
| 좋은 액션 (구체적·검증 가능) | "PoC 데모 미팅 진행", "계약 조건 최종 합의 미팅" |

---

## CA — 전화 (Call)

**Example 1**
콜 발화: "다음 주 화요일에 기술 검토 결과 다시 통화하기로 했어요"

```json
{
  "inquiry": "<inquiry_id>",
  "type": "CA",
  "dueDate": "2026-05-26",
  "dueTime": "14:00",
  "action": "기술 검토 결과 콜백",
  "actionHistory": "고객 측 기술팀과 검토 후 결과 공유 약속"
}
```

**Example 2**
콜 발화: "제안서 보내고 나서 목요일에 전화로 질문 받기로 했어요"
Todo body:
```json

  "inquiry": "<inquiry_id>",
  "type": "CA",
  "dueDate": "2026-05-22",
  "action": "제안서 관련 질의응답 콜",
  "actionHistory": "제안서 발송 후 궁금한 점 확인 목적"
}
```

| 구분 | 예시 |
|------|------|
| 나쁜 액션 (지나치게 추상적) | "전화하기", "연락하기", "확인 콜" |
| 좋은 액션 (구체적·검증 가능) | "기술 검토 결과 콜백", "제안서 관련 질의응답 콜" |

## 공통 작성 원칙

1. **구체적 목적**: 무엇을, 누구에게, 어떤 형태로 → 한 줄로 표현
2. **검증 가능**: 완료 여부를 객관적으로 판단할 수 있는 액션
3. `actionHistory`: 콜에서 나온 실제 맥락만 기록, 추측 금지
4. `dueDate`: 상대 표현("다음 주 화요일")은 절대 날짜로 변환 후 사용자 확인
5. `dueTime`: 특정 시각이 합의된 경우에만 포함
