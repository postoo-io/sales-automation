# Slack 채널 매칭 룰

call-prep 스킬이 고객사 Slack 채널을 찾을 때 사용하는 휴리스틱과 규칙을 정의합니다.

---

## 회사명 정규화 룰

채널 검색 전 회사명을 아래 순서로 정규화합니다.

1. 법인 접두·접미어 제거: `(주)`, `주식회사`, `Inc.`, `Co., Ltd.`, `Corp.`, `Ltd.`
2. 앞뒤 공백 제거
3. 내부 공백 → 하이픈(`-`) 치환
4. 전체 소문자 변환

예시: `(주) 에이스 테크 Inc.` → `에이스-테크`

---

## 매칭 우선순위

| 순위 | 조건 | 동작 |
|------|------|------|
| 1 | 정규화된 회사명과 채널명이 **정확히 일치** | 즉시 사용 |
| 2 | 채널명에 정규화된 회사명이 **포함** | 후보로 수집 |
| 3 | 채널명에 **의뢰명·프로젝트명 키워드** 포함 | 후보로 수집 |
| 4 | (대안) `mcp__slack__search_messages`로 회사 도메인 포함 메시지가 다수인 채널 | 사용자에게 제안 |

---

## 모호한 케이스 처리

| 매칭 결과 | 처리 방법 |
|-----------|-----------|
| 0개 | "Slack에서 채널을 찾지 못했습니다. 워크스페이스에서 채널명을 직접 확인 후 알려 주세요." 표시 |
| 1개 | 자동으로 해당 채널 사용 |
| 2개 이상 | 채널 목록을 사용자에게 제시하고 선택 요청 (임의 선택 금지) |

---

## 자동 가입 금지

- 봇이 채널에 없으면 메시지에 접근할 수 없습니다.
- 이 경우 사용자에게 알림: "해당 채널에 봇이 없어 메시지를 읽을 수 없습니다. 채널에 봇을 초대한 후 다시 시도해 주세요."
- 봇을 채널에 **자동으로 가입시키는 행위는 금지**합니다. 반드시 사용자가 직접 초대해야 합니다.

---

## 민감 키워드 (landmine 감지 후보)

채널의 최근 메시지에서 아래 키워드가 발견되면 심층 브리프의 **지뢰·주의사항** 섹션에 노출합니다.

| 키워드 | 의미 |
|--------|------|
| `리스크` | 위험 요소 언급 |
| `우려` | 불안·걱정 표현 |
| `이의제기` | 공식 이의 |
| `경쟁사` | 경쟁 상황 |
| `가격` | 가격 협상·불만 |
| `클레임` | 공식 클레임 |
| `해지` | 계약 해지 논의 |
| `유보` | 결정 보류 |
| `대기` | 진행 중단 |
| `중단` | 작업·프로세스 중단 |
| `보류` | 진행 보류 |

감지된 키워드가 없으면 지뢰·주의사항 섹션에 "감지된 위험 신호 없음"을 표시합니다.
