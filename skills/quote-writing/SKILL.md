# 견적서 작성 (Quote Writing)

Pluuug 견적 항목 라이브러리에서 단가를 조회하고, 견적 초안을 계산·출력한다. 손익 분석·리스크 감지·과거 견적 비교를 포함하며, 사용자 승인 후 Pluuug 의뢰 갱신 및 Drive 저장까지 수행한다.

---

## 1. 트리거 예시

| 발화 유형 | 예시 문장 |
|---|---|
| 신규 작성 | "ABC 컴퍼니 견적서 작성해줘" |
| 특정 의뢰 지정 | "의뢰 ID 1042 견적 써줘 — 서버 구축 3개월" |
| 단가 계산 | "도입 5명, 1년 라이선스 단가 계산해줘" |
| 수정 | "10% 할인 적용해서 다시 계산해줘" |
| 손익만 | "이 견적 손익 분석해줘" |
| 리스크만 | "이 견적 리스크만 봐줘" |
| 과거 비교 | "지난번 견적이랑 비교해줘" |
| 빠른 저장 | "PDF만 빨리 Drive에 올려줘" |

---

## 2. 입력

| 항목 | 설명 | 필수 여부 |
|---|---|---|
| 대상 | Pluuug 의뢰 ID 또는 고객 ID. 없으면 회사명 + 담당자명 | 권장 |
| 품목 목록 | 항목명 / 수량 / 단가 / 단위 (M/M, EA, 식, 월, 년) | 필수 |
| 기간 / 계약 조건 | 시작일, 종료일, 월 단위 / 연 단위 | 상황별 |
| 할인 | 정액 또는 퍼센트 (예: 10%, 500,000원) | 선택 |
| 부가세 유형 | E(별도) / I(포함) / N(없음). 미입력 시 기본값 E 적용 | 선택 |
| 유효기간 | 기본값: 발행일 + 14일 | 선택 |
| 외주비 / SaaS 비용 | 손익 분석 시 원가 계산에 필요 | 손익 분석 시 |

빠진 값은 **한 번에 모아서 사용자에게 묻는다**. 항목별로 반복 질문 금지.

---

## 3. 절차

### 단계 1 — 템플릿 매칭

```bash
# 분류(카테고리) 목록 조회
scripts/pluuug.py GET /v1/estimate/item/classification

# 사용자 발화 키워드로 단가 후보 검색 (키워드마다 반복)
scripts/pluuug.py GET /v1/estimate/item --query search=<키워드> page_size=50
```

- 응답 항목의 `title` / `unit` / `unitCost` / `classification`을 사용자 발화 품목과 부분 매칭한다.
- **신뢰도 낮은 매칭** (제목 유사도 < 70%) → 사용자에게 "이 항목이 맞나요?" 확인.
- **매칭 안 되는 항목** → "신규 단가 후보"로 분리 → 사용자에게 단가 근거 요청.
- 고객 정보 보강: `scripts/pluuug.py GET /v1/client/{id}` 또는 의뢰의 `client` 필드.

손익 계산은 `scripts/calc_margin.py`, 리스크 감지는 `scripts/detect_risks.py`로 자동화합니다 (stdin/stdout JSON 파이프).

### 단계 2 — 견적 계산

```
소계(항목) = 수량 × 단가
할인액     = 소계합계 × 할인율  (또는 정액)
과세표준   = 소계합계 - 할인액
부가세     = 과세표준 × 0.1     (vatType=E 일 때)
합계       = 과세표준 + 부가세  (vatType=E)
합계       = 소계합계 - 할인액  (vatType=I, 부가세 포함)
합계       = 소계합계 - 할인액  (vatType=N, 부가세 없음)
```

- **누적 할인 금지**: 할인은 단 한 번만 적용. 중복 적용 위험 시 경고 출력.
- 모든 중간값(소계·할인액·과세표준·부가세)을 출력에 포함한다.
- 부가세 포함/별도 표기는 모든 출력에 반드시 명시.

### 단계 3 — 손익 분석

```bash
# 실무자 인건비 단가 조회 (manMonthAmount 필드 활용)
scripts/pluuug.py GET /v1/worker
```

**원가 추정 방법 (우선순위 순)**

1. 사용자가 외주비/SaaS 비용을 직접 제공 → 그대로 사용.
2. 사용자가 투입 M/M을 알면 `원가 = manMonthAmount × M/M`.
3. 실무자 정보 없는 항목 → "이 항목의 원가(외주비·라이선스 등)를 알려주세요" 요청.
4. 정보 미제공 시 해당 항목 원가를 `?`로 표시하고 마진율 계산 제외.

**손익 지표**

```
항목 마진    = 항목 소계 - 항목 원가
항목 마진율  = 항목 마진 / 항목 소계 × 100 (%)
영업이익     = 견적 합계(부가세 제외) - 총원가
영업이익률   = 영업이익 / 견적 합계(부가세 제외) × 100 (%)
```

**임계치**

| 마진율 | 표시 |
|---|---|
| ≥ 20% | 정상 |
| 10% ~ 20% 미만 | ⚠️ 주의 — 마진율 낮음 |
| 10% 미만 | 🚨 위험 — 손익 검토 필수 |

> **원가 데이터는 내부 화면에만 표시**. 견적 PDF, 메일, Pluuug text_history에 마진율/원가를 절대 기재하지 않는다.

### 단계 4 — 과거 견적 비교

```bash
# 동일 고객의 과거 계약/견적 이력 조회
scripts/pluuug.py GET /v1/contract --query client_id=<id> end_date_from=2024-01-01

# 의뢰별 text_history에서 "견적" 키워드 확인
scripts/pluuug.py GET /v1/inquiry/{id}
```

- text_history 중 "견적" 키워드가 포함된 메모를 추출해 직전 견적 정보를 파악한다.
- 이번 견적과 항목·단가·할인·스코프를 비교해 아래 형식으로 출력한다.

```
| 항목 | 이번 단가 | 직전 단가 | 변동 | 비고 |
|---|---:|---:|---:|---|
| 서버 구축 | 5,000,000 | 4,500,000 | +11.1% | 범위 확대 |
```

### 단계 5 — 리스크 감지

아래 규칙을 순서대로 점검하고, 해당되는 항목을 표로 출력한다.

| # | 리스크 항목 | 감지 조건 | 영향 | 권고 |
|---|---|---|---|---|
| R1 | 마진율 낮음 (주의) | 영업이익률 < 20% | 수익성 저하 | 단가 인상 또는 할인 축소 검토 |
| R2 | 마진율 위험 | 영업이익률 < 10% | 적자 가능성 | 견적 재검토 전까지 발행 보류 권고 |
| R3 | 신규 단가 비중 과다 | 라이브러리 미등록 항목 소계 / 전체 소계 > 30% | 단가 신뢰도 저하 | 단가 근거 재확인 후 라이브러리 등록 |
| R4 | 과도한 할인 | 할인율 > 15% | 정책 이탈 / 가격 기준 훼손 | 승인 권한자 검토 필요 |
| R5 | 가격 일관성 문제 | 동일 client에 30일 내 다른 견적 존재 | 혼선·신뢰도 하락 | 기존 견적 무효화 여부 확인 |
| R6 | 부가세 표기 불명확 | vatType 미입력 / 출력에 명시 없음 | 계약 분쟁 리스크 | vatType 명시 후 재발행 |
| R7 | 유효기간 촉박 | 유효기간 < 7일 | 고객 검토 시간 부족 | 유효기간 연장 권고 |
| R8 | 미확정 신호 | 의뢰 본문/text_history에 "예산 미상", "검토 중", "미정" 포함 | 수주 불확실 | 예산 확정 후 견적 발행 권고 |
| R9 | 결제 조건 미합의 | 선금/잔금 조건 언급 없음 | 대금 회수 리스크 | 결제 조건 비고란 명시 |
| R10 | 외부 라이선스 원가 누락 | SaaS/외부 라이선스 항목 존재하나 원가에 미반영 | 손익 과대 계산 | 해당 항목 원가 재입력 요청 |

리스크가 없으면 "감지된 리스크 없음"을 출력한다.

### 단계 6 — 출력 (마크다운)

모든 출력은 사용자 승인 전까지 **초안**으로 표시한다.

#### 6-A. 견적서 본문

```markdown
## 견적서 초안 — [회사명] / [담당자]

| 항목 | 내용 |
|---|---|
| 발행일 | 2026-05-21 |
| 유효기간 | 2026-06-04 |
| 공급자 | (자사명) |
| 수신자 | [회사명] [담당자] |
| 부가세 | 별도 (E) |

| # | 항목 | 단위 | 수량 | 단가 | 소계 |
|---|---|---|---:|---:|---:|
| 1 | 서버 구축 | 식 | 1 | 5,000,000 | 5,000,000 |
| 2 | 유지보수 | 월 | 3 | 800,000 | 2,400,000 |

| 구분 | 금액 |
|---|---:|
| 소계 | 7,400,000 |
| **할인 (10%)** | **-740,000** |
| 과세표준 | 6,660,000 |
| 부가세 (10% 별도) | 666,000 |
| **합계** | **7,326,000** |

비고: 유지보수 기간 3개월, 결제 조건 미정
```

#### 6-B. 손익 분석 (내부용)

```markdown
## 손익 분석 (내부용 — 외부 공유 금지)

| # | 항목 | 소계 | 원가 | 마진 | 마진율 |
|---|---|---:|---:|---:|---:|
| 1 | 서버 구축 | 5,000,000 | 3,500,000 | 1,500,000 | 30.0% |
| 2 | 유지보수 | 2,400,000 | 1,800,000 | 600,000 | 25.0% |
| — | **합계** | **7,400,000** | **5,300,000** | **2,100,000** | **28.4%** |

영업이익 (할인 후): 2,100,000 - 740,000 = **1,360,000원**
영업이익률: **20.4%** ✅ 정상
```

#### 6-C. 과거 견적 비교

```markdown
## 과거 견적 비교

| 항목 | 이번 단가 | 직전 단가 | 변동 | 비고 |
|---|---:|---:|---:|---|
| 서버 구축 | 5,000,000 | 4,500,000 | +11.1% | 범위 확대 반영 |
| 유지보수 | 800,000 | 800,000 | 0% | 동일 |
```

#### 6-D. 리스크 표

```markdown
## 리스크 감지 결과

| 리스크 | 근거 | 영향 | 권고 |
|---|---|---|---|
| R9 결제 조건 미합의 | 비고란에 결제 조건 없음 | 대금 회수 리스크 | 결제 조건 비고란 명시 |
```

#### 6-E. 추천 제안

- 마진율이 낮은 경우: "X 항목 단가를 10% 인상하면 영업이익률 24.5%로 개선됩니다."
- 할인이 과도한 경우: "할인을 10%로 낮추면 R4 리스크가 해소됩니다."
- 신규 단가 비중이 높은 경우: "라이브러리 미등록 단가 비중이 35%입니다. 확인 후 등록을 권장합니다."

### 단계 7 — 저장 (사용자 승인 후에만 실행)

사용자가 "발행" 또는 "저장"을 확인한 후 아래 순서로 진행한다.

```bash
# 1. 의뢰의 예상 견적 금액 갱신
scripts/pluuug.py PATCH /v1/inquiry/{id} --json '{"estimate":"<합계>"}'

# 2. 견적 발행 메모 기록 (마진율·원가는 절대 포함 금지)
scripts/pluuug.py POST /v1/inquiry/{id}/text_history \
  --json '{"content":"견적 v1 발행 — 합계 7,326,000원(부가세 별도), 유효 2026-06-04, 할인 10%"}'

# 3. 신규 단가 항목이 있으면 사용자 확인 후 라이브러리 등록
scripts/pluuug.py POST /v1/estimate/item \
  --json '{"title":"신규항목명","unitCost":"단가","unit":"식"}'
```

**Drive 저장**

- `google_drive`로 `/영업/2026/견적서/` 폴더에 마크다운 또는 PDF 업로드.
- 폴더가 없으면 **임의로 생성하지 않고** 사용자에게 확인 후 생성.
- 파일명 형식: `견적서_[회사명]_[YYYYMMDD]_v1.md`

## 4. 손익 분석 룰 상세

### 원가 추정 우선순위

1. **사용자 직접 입력**: 외주비, SaaS 비용, 라이선스 비용 등.
2. **인건비 계산**: `GET /v1/worker` 응답의 `manMonthAmount` × 투입 M/M.
3. **정보 없는 항목**: 원가 `?` 처리, 해당 항목을 손익 계산에서 제외, 사용자에게 입력 요청.

### 임계치 요약

| 영업이익률 | 판정 | 조치 |
|---|---|---|
| ≥ 20% | ✅ 정상 | — |
| 10% ~ 19.9% | ⚠️ 주의 | 재검토 권고, 발행은 가능 |
| < 10% | 🚨 위험 | 발행 전 재검토 강력 권고 |

### 원가 노출 금지 원칙

견적 PDF, 메일 본문, Pluuug text_history, Slack 메시지 **어디에도 원가·마진율을 기재하지 않는다**. 손익 분석은 내부 화면 출력에만 표시한다.

---

## 5. 리스크 감지 룰 요약

R1–R10 총 10개 룰을 `scripts/detect_risks.py`가 자동 점검한다.

| 코드 | 리스크 | 심각도 |
|---|---|---|
| R1 | 마진율 낮음 / 위험 | warning / critical |
| R2 | 신규 단가 비중 > 30% | warning |
| R3 | 할인율 > 15% | warning |
| R4 | 30일 내 동일 client 다른 견적 | warning |
| R5 | 부가세 vatType 불명확 | critical |
| R6 | 유효기간 < 7일 | warning |
| R7 | 미확정 키워드 감지 | warning |
| R8 | 결제 조건 미합의 | warning |
| R9 | SaaS/라이선스 원가 누락 | critical |
| R10 | 전체 항목 신규 단가 (100%) | critical |

전체 룰과 임계치 근거는 `references/risk_catalog.md` 참조.

## 6. 출력 형식 템플릿

출력은 항상 아래 순서로 구성한다. 변형 요청 시 해당 섹션만 출력한다.

```
1. 견적서 본문 (회사·담당자·발행일·유효기간·항목표·소계·할인·부가세·합계·비고)
2. 손익 분석 표 (내부용 — 외부 공유 금지 워터마크 포함)
3. 과거 견적 비교 표
4. 리스크 감지 표
5. 추천 제안 (불릿 형식)
6. 액션 확인 ("이대로 발행할까요? 발행 / 수정 / 취소")
```

고객 발송용 견적서는 `assets/quote_template.md` 템플릿을 사용 (마진 변수는 외부 노출 금지).

## 7. 변형 요청 응대

| 요청 유형 | 동작 |
|---|---|
| "10% 할인 다시" | 직전 견적 변수 유지, 할인만 변경, 손익·리스크 재계산 후 전체 출력 |
| "지난번 견적이랑 비교" | 과거 견적 비교 표(6-C)만 출력 |
| "리스크만 봐줘" | 리스크 표(6-D) + 추천 제안(6-E)만 출력 |
| "손익 분석만" | 손익 분석 표(6-B)만 출력 |
| "PDF만 빨리" | 초안 확인 후 Drive 저장까지 진행. 의뢰 estimate 갱신은 "업데이트할까요?" 별도 확인 |
| "견적 템플릿 보여줘" | `GET /v1/estimate/item/classification` + `GET /v1/estimate/item` 결과를 표로 출력 |
| "수정해줘 — [내용]" | 해당 항목만 변경, 나머지 변수 유지, 전체 재계산 후 출력 |

## 8. 가드레일

- **금액·할인 확정 금지**: 초안 출력은 자유. 저장·갱신 API 호출은 반드시 사용자 명시적 승인 후.
- **단가 0 또는 공백**: 발행 금지. 사용자에게 단가 입력 요청.
- **30일 내 기존 견적 존재**: "기존 견적 수정 / 새 견적 발행" 중 무엇인지 반드시 확인.
- **신규 단가 라이브러리 추가**: 사용자 확인 후에만 `POST /v1/estimate/item` 호출.
- **부가세 표기**: 모든 출력에 E(별도)/I(포함)/N(없음) 중 하나를 반드시 명시.
- **원가/마진율 외부 노출 금지**: PDF·메일·Pluuug text_history·Slack에 기재 금지.
- **누적 할인 금지**: 할인은 소계 합계에 단 1회만 적용. 항목별 할인 + 전체 할인 중복 적용 금지.
- **Drive 폴더 자동 생성 금지**: 폴더 없으면 사용자 확인 후 생성.
- **Rate limit**: Pluuug Open API 1,000 req/min 준수. 키워드 검색 반복 시 순차 호출.
- **견적서 자체 발행 API 없음**: Pluuug Open API는 견적서 PDF 직접 발행을 지원하지 않는다 (2026-05 기준). 항목 템플릿 CRUD와 의뢰 estimate 합계 갱신만 가능. 실제 견적서 발행은 Drive 저장 또는 Pluuug 대시보드를 이용한다.
